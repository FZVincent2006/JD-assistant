# 飞书 JD 公司标题自动编号设计

## 目标

在“岗位JD整理”区域为全新公司创建带飞书自动编号的一级标题。新公司插入区域最前面后显示为 `1. 公司名`，后续公司由飞书自动顺延；编号不写入公司名文本。

## 范围

- 仅修改飞书 OpenAPI 模式中新公司 JD 公司标题的生成、模板识别和写入后校验。
- Portfolio 区格式保持不变。
- 老公司追加岗位不创建或修改公司标题。
- Boss、脉脉及原有单岗位流程保持不变。

## 技术设计

飞书 Docx OpenAPI 的标题与列表共用 `TextStyle`。`style.sequence` 用于控制有序编号，值为 `"auto"` 时由飞书自动生成并连续重排编号。

1. 模板读取时继续从真实 JD 公司一级标题复制字体、链接和段落样式。
2. 新公司渲染时保留一级标题 Block（`block_type: 3`），并强制设置 `heading1.style.sequence` 为 `"auto"`。
3. 公司名内容只写公司名，不添加 `1.` 等序号文本。
4. 写入后重新读取新公司块，除原有标题、位置和结构校验外，再校验：
   - 块类型为 Heading 1；
   - `heading1.style.sequence === "auto"`。
5. 追加岗位模式不渲染公司标题，因此不会改变现有编号。

## 兼容与失败策略

- 即使历史样例标题缺少 `sequence`，新创建的公司标题也明确补为 `"auto"`，避免依赖不完整模板。
- 如果飞书回读的新公司标题没有自动编号样式，JD 区校验失败并停止后续写入，不把编号文本作为降级方案。
- 不改变 Portfolio、Boss、脉脉模块的消息、解析或写入路径。

## 测试

- 渲染测试：新公司 Heading 1 含 `sequence: "auto"`，文本不含 `1.`。
- 模板测试：复制其他样式的同时正确保留或规范自动编号。
- 追加测试：`append-jobs` 不产生公司标题。
- 校验测试：接受正确自动编号，拒绝缺失或固定编号值。
- 完整测试套件、生产构建以及 Boss/脉脉完整性回归全部通过后再同步到测试扩展目录。

## 依据

飞书官方 Go SDK 中 `TextStyle.Sequence` 的定义为“用于确定有序列表项编号，为具体数值或 `auto`”。
